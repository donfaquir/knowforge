# 模块化编码守则

## 核心原则

**禁止向已膨胀的根级文件追加新代码。** `src/App.tsx` 和 `src/App.css` 已经过度膨胀，后续开发中，除非修改已有逻辑，否则不得向这两个文件新增功能代码或样式。

## 具体规则

### 1. TypeScript / TSX

- 新增功能组件必须放在 `src/components/` 下独立文件中。
- 新增 Hook 必须放在 `src/hooks/` 下独立文件中。
- 新增 Context 必须放在 `src/contexts/` 下独立文件中。
- `src/App.tsx` 仅允许：导入和组装顶层组件、修改已有路由/布局逻辑。不允许在其中新增业务函数或事件处理器。

### 2. CSS

- 新增组件的样式必须使用与组件同名的独立 CSS 文件（如 `MyComponent.css`），并在组件文件中 `import`。
- `src/App.css` 仅允许：修改已有样式、全局变量（`:root`）、全局 reset。不允许在其中新增组件级样式。
- 暗色模式样式应跟随组件的独立 CSS 文件，使用 `@media (prefers-color-scheme: dark)` 在同文件内定义。

### 3. 判断标准

当你需要添加新代码时，先问自己：
- 这段代码是对 `App.*` 中已有逻辑的修改吗？ → 可以编辑 `App.*`
- 这是一个新功能/新组件/新样式？ → **必须** 创建独立模块文件

### 4. 例外情况

如果确实需要在 `App.*` 中添加内容（如全局事件监听、顶层 Provider 包裹），必须在 commit message 或注释中说明原因。
